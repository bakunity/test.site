Техническое задание на разработку сайта Pro-plitka
1. Общая информация
1.1. Название проекта
Pro-plitka
1.2. Тип проекта
Корпоративный сайт / сайт-презентация компании по укладке плитки.
1.3. Основная задача сайта
Создать сильное онлайн-представительство компании Pro-plitka, которое формирует доверие, демонстрирует уровень работ, показывает экспертность команды и помогает потенциальным заказчикам оставить заявку.
Сайт должен выглядеть как премиальный бренд, а не как обычная строительная страница.
Главный эффект после просмотра сайта:
“Эта команда выглядит надёжно, профессионально и дорого. Хочу, чтобы они работали на моём объекте.”
2. Описание компании
Pro-plitka занимается укладкой плитки любого вида и любой сложности.
Компания работает:
с частными клиентами;
с юридическими лицами;
с коммерческими объектами;
с крупными объёмами;
с государственными объектами;
с администрациями;
со школами;
с детскими садами;
с домами культуры;
с объектами под ключ;
с дизайнерскими решениями.
Компания выполняет плиточные работы на:
полу;
стенах;
фасадах;
санузлах;
коммерческих помещениях;
общественных зданиях;
технических помещениях;
объектах с повышенными требованиями к качеству.
3. Целевая аудитория
3.1. Частные клиенты
Люди, которым нужна качественная укладка плитки в квартире, доме, санузле, ванной, кухне, на полу или стенах.
3.2. Коммерческие заказчики
Компании, застройщики, подрядчики, владельцы помещений, которым нужны плиточные работы на больших объектах.
3.3. Государственные и муниципальные заказчики
Администрации, школы, сады, дома культуры и другие учреждения, которым нужен подрядчик с понятной репутацией и презентабельным портфолио.
3.4. Партнёры и подрядчики
Дизайнеры, архитекторы, строительные компании, ремонтные бригады и генподрядчики, которым нужен надёжный исполнитель по плитке.
4. Цели сайта
4.1. Репутационная цель
Сформировать образ компании как сильного, аккуратного, премиального и надёжного подрядчика.
4.2. Коммерческая цель
Получать заявки с сайта через форму, которая отправляет данные в Telegram.
4.3. Презентационная цель
Дать возможность быстро отправлять потенциальным клиентам ссылку на сайт, портфолио или PDF-презентацию.
4.4. Техническая цель
Создать быстрый, стабильный, адаптивный сайт без VPS, с простой структурой, удобной для дальнейших правок через Codex или AI-агентов.
5. Требования к стилю сайта
5.1. Визуальное направление
Стиль сайта:
Linear + Stripe, но в Dark Premium-версии.
Сайт должен быть:
тёмным;
дорогим;
контрастным;
современным;
технологичным;
чистым;
уверенным;
не перегруженным;
с качественной типографикой;
с плавными градиентами;
с аккуратными карточками;
с премиальными акцентами.
5.2. Цветовое направление
Базовые цвета:
тёмный фон: графитовый / почти чёрный;
светлый текст: белый / молочный;
вторичный текст: серый;
акценты: синий, стальной, золотистый, оранжевый или холодный premium-gradient;
карточки: тёмные с лёгкой обводкой;
кнопки: контрастные, заметные, премиальные.
Финальные цвета можно изменить после первого прототипа.
5.3. Общее ощущение
Сайт должен ощущаться как сайт серьёзной компании, которая работает с крупными объектами, умеет вести процессы и отвечает за качество.
Не должно быть ощущения дешёвого ремонта, шаблонного лендинга или перегруженного строительного сайта.

7. Архитектура проекта
7.1. Общая схема
Пользователь открывает сайт:
Пользователь
  -> https://your-domain.com
  -> форма заявки
  -> Cloudflare Pages Function
  -> Telegram Bot API
  -> сообщение владельцу в Telegram
7.2. Архитектурная схема
GitHub repo: pro-plitka
  -> Astro сайт
  -> Tailwind CSS компоненты
  -> Контент в JSON/Markdown/YAML/TS
  -> Cloudflare Pages
  -> Домен
  -> Cloudflare Pages Function
  -> Telegram Bot API
  -> Заявка в Telegram
7.3. Почему без VPS
Проект должен работать без отдельного сервера.
Все статические страницы, картинки, PDF и frontend размещаются на Cloudflare Pages.
Форма заявки обрабатывается через Cloudflare Pages Functions.

11. Логика формы заявки
11.1. Что происходит после отправки
Пользователь заполняет форму.
Форма отправляет POST-запрос в Cloudflare Pages Function.
Function отправляет сообщение через Telegram Bot API.
Владелец получает сообщение в Telegram.
11.2. Пример сообщения в Telegram
🟢 Новая заявка с сайта Pro-plitka

Имя: Иван
Телефон: +7...
Telegram / WhatsApp: @ivan
Сообщение: Хочу консультацию по объекту

Страница: /services
Источник: сайт
UTM: utm_source=...
Выбранная услуга: Укладка керамогранита
Время: 2026-06-03 14:30
11.3. Требования к форме
Форма должна:
проверять обязательные поля;
не отправляться пустой;
показывать статус отправки;
показывать успешное сообщение;
показывать ошибку, если Telegram API недоступен;
защищаться от спама простым honeypot-полем;
не раскрывать токен Telegram-бота на клиенте.

14. Cloudflare Pages Function
14.1. Endpoint
POST /api/contact
14.2. Назначение
Принимать заявку с формы и отправлять её в Telegram.
14.3. Переменные окружения
В Cloudflare должны быть добавлены переменные:
TELEGRAM_BOT_TOKEN
TELEGRAM_CHAT_ID
SITE_NAME
14.4. Локальный пример
Файл .env.example:
TELEGRAM_BOT_TOKEN=your_bot_token
TELEGRAM_CHAT_ID=your_chat_id
SITE_NAME=Pro-plitka
14.5. Требования безопасности
токен Telegram-бота не должен попадать в frontend;
токен хранить только в переменных окружения Cloudflare;
не коммитить .env;
добавить .env и .dev.vars в .gitignore;
использовать простую защиту от спама;
валидировать данные на стороне функции.
